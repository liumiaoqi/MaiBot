1克隆仓库与目录结构

1.1 克隆仓库

git clone https://github.com/MaiM-with-u/MaiBot.git
cd MaiBot

1.2 创建必要的目录结构

克隆下来的仓库没有 data/ 和 docker-config/ 目录，需要手动创建：

mkdir -p docker-config/mmc
mkdir -p docker-config/napcat
mkdir -p data/MaiMBot/plugins
mkdir -p data/MaiMBot/emoji
mkdir -p data/MaiMBot/logs
mkdir -p data/qq

1.3 最终目录结构

MaiBot/
├── docker-compose.yml          # Docker编排文件（需修改）
├── docker-entrypoint-wrapper.sh # 自定义入口脚本（需创建）
├── docker-config/
│   ├── mmc/                    # MaiBot核心配置
│   │   ├── bot_config.toml     # （首次运行自动生成，需修改）
│   │   └── model_config.toml   # （首次运行自动生成，需修改）
│   └── napcat/                 # NapCat配置（首次运行自动生成）
├── data/
│   ├── MaiMBot/
│   │   ├── plugins/            # 插件目录
│   │   ├── emoji/              # 表情包
│   │   └── logs/               # 日志
│   └── qq/                     # QQ登录态数据
└── depends-data/               # 运行时资源（仓库自带）

2Docker Compose 配置

2.1 修改 docker-compose.yml

需要对原始文件做两处关键修改：

❌ 原始版本（有问题）

services:
  core:
    image: sengokucola/maibot:latest
    environment:
      - TZ=Asia/Shanghai
      - EULA_AGREE=...
    ports:
      - "18001:8001"
    volumes:
      - ./docker-config/mmc:/MaiMBot/config
      - ./data/MaiMBot:/MaiMBot/data
      - ./data/MaiMBot/emoji:/data/emoji
      - ./data/MaiMBot/plugins:/MaiMBot/plugins
      - ./data/MaiMBot/logs:/MaiMBot/logs
      - ./depends-data:/MaiMBot/depends-data
    restart: always

✅ 修改后版本

services:
  core:
    image: sengokucola/maibot:latest
    entrypoint: ["/bin/sh", "/MaiMBot/docker-entrypoint-wrapper.sh"]
    environment:
      - TZ=Asia/Shanghai
      - EULA_AGREE=...
    ports:
      - "18001:8001"
    volumes:
      - ./docker-config/mmc:/MaiMBot/config
      - ./data/MaiMBot:/MaiMBot/data
      - ./data/MaiMBot/emoji:/data/emoji
      - ./data/MaiMBot/plugins:/MaiMBot/plugins
      - ./docker-entrypoint-wrapper.sh:/MaiMBot/docker-entrypoint-wrapper.sh:ro
      - ./data/MaiMBot/logs:/MaiMBot/logs
      - ./depends-data:/MaiMBot/depends-data:ro
    restart: always

2.2 修改说明

  • 添加 entrypoint：使用自定义入口包装脚本，解决插件ID冲突（见下一步）
  • 挂载 wrapper 脚本：以只读模式挂载到容器内
  • depends-data 改为只读：防止容器意外修改运行时资源

3解决插件ID冲突（关键）

这是最容易踩坑的问题！如果不处理，会导致所有插件加载失败。

3.1 问题原因

Docker镜像的入口脚本 docker-entrypoint.sh 每次启动时，会自动从模板复制 MaiBot-Napcat-Adapter 到 /MaiMBot/plugins/。如果你后续通过 WebUI 安装了 maibot-team_napcat-adapter（同ID不同目录名），IPC插件系统检测到重复插件ID后会拒绝启动整个插件运行时，导致所有插件无法加载。

3.2 解决方案：创建入口包装脚本

在项目根目录创建 docker-entrypoint-wrapper.sh：

#!/bin/sh
# MaiBot 自定义入口包装脚本
# 解决：docker-entrypoint.sh 自动从模板复制 MaiBot-Napcat-Adapter，
# 而 WebUI 安装的 maibot-team_napcat-adapter 与之同ID，
# 导致 IPC 插件系统检测到重复插件ID后拒绝启动整个插件运行时。

ADAPTER_A="/MaiMBot/plugins/MaiBot-Napcat-Adapter"
ADAPTER_B="/MaiMBot/plugins/maibot-team_napcat-adapter"

if [ -d "$ADAPTER_A" ] && [ -d "$ADAPTER_B" ]; then
    echo "[wrapper] 检测到重复 napcat-adapter 插件，删除镜像自带版本: $ADAPTER_A"
    rm -rf "$ADAPTER_A"
fi

exec /MaiMBot/docker-entrypoint.sh "$@"
此脚本会在原入口脚本执行前检测去重：如果两个 napcat-adapter 同时存在，删除镜像自带的版本（保留你通过 WebUI 安装的、有配置文件的版本）。

4首次启动与NapCat登录

4.1 启动所有容器

docker compose up -d

4.2 登录QQ（NapCat）

  1. 浏览器访问 NapCat WebUI：http://localhost:6099
  2. 使用 webui.json 中的 token 登录（首次启动后自动生成，可在 docker-config/napcat/webui.json 中查看）
  3. 扫码登录你的QQ机器人账号
  4. 登录成功后，记住你的QQ号，后续配置需要用到

4.3 配置NapCat的OneBot11 WebSocket

QQ号登录后，NapCat会自动生成 docker-config/napcat/onebot11_{QQ号}.json。需要确保其中配置了WebSocket Server供MaiBot连接：

{
  "network": {
    "httpServers": [],
    "httpSseServers": [],
    "httpClients": [],
    "websocketServers": [
      {
        "enable": true,
        "name": "maibot",
        "host": "0.0.0.0",
        "port": 3001,
        "reportSelfMessage": false,
        "messagePostFormat": "array",
        "token": "",
        "heartInterval": 30000
      }
    ],
    "websocketClients": [],
    "plugins": []
  }
}
端口 3001 是MaiBot连接NapCat的默认端口，确保与后续 adapter 配置一致。

5模型配置 (model_config.toml)

文件docker-config/mmc/model_config.toml

首次启动后MaiBot会自动生成此文件，但API Key是占位符，必须修改。

5.1 配置 API Provider

默认配置了DeepSeek，但API Key是 "your-api-key"。你需要替换为真实Key，并按需添加更多Provider。

推荐配置两个Provider：

[[api_providers]]
name = "BaiLian"                          # DeepSeek API（命名随意，models中引用此名）
base_url = "https://api.deepseek.com"
api_key = "sk-你的DeepSeek密钥"            # ← 替换！
client_type = "openai"
max_retry = 2
timeout = 30

[[api_providers]]
name = "Alibaba"                          # 阿里云百炼（通义千问兼容API）
base_url = "https://dashscope.aliyuncs.com/compatible-mode/v1"
api_key = "sk-你的阿里云百炼密钥"          # ← 替换！
client_type = "openai"
max_retry = 2
timeout = 30

5.2 配置模型列表

# 聊天模型（DeepSeek）
[[models]]
model_identifier = "deepseek-v4-flash"
name = "deepseek-v4-flash"
api_provider = "BaiLian"
price_in = 0
price_out = 0
visual = false

[[models]]
model_identifier = "deepseek-v4-pro"
name = "deepseek-v4-pro"
api_provider = "BaiLian"
price_in = 0
price_out = 0
visual = false

# 视觉模型（阿里云百炼 - Qwen）
[[models]]
model_identifier = "qwen3.6-max-preview"
name = "qwen3.6-max-preview"
api_provider = "Alibaba"
price_in = 0
price_out = 0
visual = true

# Embedding模型（阿里云百炼）
[[models]]
model_identifier = "text-embedding-v4"
name = "text-embedding-v4"
api_provider = "Alibaba"
price_in = 0
price_out = 0
visual = false

5.3 配置任务模型分配

[model_task_config.replyer]
model_list = ["deepseek-v4-flash", "deepseek-v4-pro"]

[model_task_config.planner]
model_list = ["deepseek-v4-flash", "deepseek-v4-pro"]

[model_task_config.utils]
model_list = ["deepseek-v4-flash", "deepseek-v4-pro"]

[model_task_config.vlm]
model_list = ["qwen3.6-max-preview"]

[model_task_config.embedding]
model_list = ["text-embedding-v4"]
Embedding模型必须配置！默认配置的 deepseek-v4-pro 不支持embedding，会导致记忆系统的向量检索完全失效（降级为关键词匹配）。推荐使用阿里云百炼的 text-embedding-v4，价格极低（约0.0007元/千token）。

文件docker-config/mmc/bot_config.toml

6.1 基础信息

[bot]

6.2 人格设定

这是Bot的灵魂，建议认真编写：

[personality]
personality = "你是XXX，...(详细人设描述，建议200字以内)"
reply_style = "【语言风格】\n语气特点：...\n常用表达：..."
multiple_reply_style = [
    "你的风格平淡但不失讽刺",
    "用1-2个字进行回复",
    "用1-2个符号进行回复",
]

6.3 聊天参数

[chat]
talk_value = 0.3              # 群聊发言频率（0-1，越小越沉默）
private_talk_value = 1        # 私聊发言频率
inevitable_at_reply = true    # AT必回
max_context_size = 40         # 上下文长度
enable_context_optimization = true  # 优化Planner上下文消耗
mid_term_memory = true        # 中期记忆摘要

6.4 WebUI安全

[webui]
enabled = true
host = "0.0.0.0"             # ← Docker部署必须改为0.0.0.0！
port = 8001                   # 与docker-compose.yml端口映射右侧一致
allowed_ips = "127.0.0.1"    # IP白名单，按需修改
Docker部署时 webui.host 必须改为 "0.0.0.0"！默认值是 "127.0.0.1"，在容器内这意味着只有容器自身能访问WebUI，宿主机无法通过端口映射访问。改完前浏览器打不开WebUI。

7NapCat-Adapter 插件配置

这是MaiBot连接QQ的桥梁，配置错误会导致Bot无法收发消息。

7.1 关键配置项

文件data/MaiMBot/plugins/MaiBot-Napcat-Adapter/config.toml

napcat_server.host 绝对不能是 "127.0.0.1"！在Docker环境中，core和napcat是不同容器，必须使用Docker容器名或网络别名通信。
[plugin]
enabled = true                # ← 必须为true！
config_version = "0.1.0"

[napcat_server]
host = "napcat"               # ← Docker Compose服务名，不是127.0.0.1！
port = 3001                   # 与NapCat onebot11配置一致
token = ""                    # 若NapCat设置了token则填写

[chat]
enable_chat_list_filter = true
group_list_type = "whitelist"
group_list = [123456789]      # ← 允许Bot活跃的群号列表
private_list_type = "whitelist"
private_list = [987654321]    # ← 允许Bot私聊的QQ号列表
群聊收不到消息？NapCat适配器默认启用聊天名单过滤，且群聊默认是白名单模式。如果群号没有写进 group_list，群消息会被适配器直接丢弃，看起来就像NapCat已连接但MaiBot没反应。一定要把你希望Bot活跃的群号加入白名单！
Docker Compose 中 napcat 的服务名是 napcat（即 docker-compose.yml 中的 service key），core容器可以通过此名称在Docker bridge网络中解析到napcat容器。不是 maim-bot-napcat（那是container_name）。

10常见问题排查

Q1: 所有插件加载失败，日志显示"检测到重复插件ID"

原因：MaiBot-Napcat-Adapter（镜像自带）与 maibot-team_napcat-adapter（WebUI安装）同ID冲突。

解决：确认 docker-entrypoint-wrapper.sh 已创建且 docker-compose.yml 中已配置 entrypoint 和 volume 挂载。重启容器。

Q2: NapCat连接失败 "Cannot connect to host 127.0.0.1:3001"

原因：adapter 配置中 host 指向了 127.0.0.1，但NapCat在另一个容器中。

解决：将 napcat_server.host 改为 "napcat"（Docker Compose服务名，即docker-compose.yml中的service key）。

Q3: "关系向量未启用" / embedding 进入降级态

原因：embedding模型配置的模型不支持embedding（如 deepseek-v4-pro）。

解决：在 model_config.toml 中将 embedding 的 model_list 改为支持embedding的模型（如阿里云百炼的 text-embedding-v4），并确保对应的 API Provider 和 model 定义存在。

Q4: 插件"配置版本非法"

原因：插件自身的 config_model 定义与运行时校验不兼容，通常是插件版本bug。

解决：在WebUI中卸载重装该插件，或等待插件作者更新。

Q5: WebUI无法下载插件

原因：GitHub在国内无法直接访问。

解决：在WebUI设置中配置GitHub镜像源。
