MaiBot 首次启动后配置修改指南

完成 docker compose up -d 后，按本指南依次修改三个配置文件

阶段 A：创建目录并首次启动

A1. 创建必要目录

cd MaiBot
mkdir -p docker-config/mmc
mkdir -p docker-config/napcat
mkdir -p data/MaiMBot/plugins
mkdir -p data/MaiMBot/emoji
mkdir -p data/MaiMBot/logs
mkdir -p data/qq

A2. 首次启动（生成配置文件）

docker compose up -d

启动后MaiBot会自动在 docker-config/mmc/ 下生成 bot_config.toml 和 model_config.toml，但里面的关键值都是占位符，必须修改。

A3. 确认文件已生成

ls docker-config/mmc/
# 应输出：bot_config.toml  model_config.toml

阶段 B：修改 model_config.toml

文件docker-config/mmc/model_config.toml

这是模型配置文件，决定了Bot用什么AI模型。首次生成的文件使用DeepSeek占位API Key，需要替换为真实Key并添加Embedding模型。

B1. 替换 API Key

查找

api_key = "your-api-key"

替换为你的 DeepSeek API Key

api_key = "sk-你的DeepSeek密钥"

API Key 必须替换，否则Bot无法调用任何AI模型，启动后会报错停止。

B2. 添加阿里云百炼 API Provider（用于 Embedding 和视觉模型）

在文件末尾的 [[api_providers]] 段之后，追加：

[[api_providers]]
name = "Alibaba"
base_url = "https://dashscope.aliyuncs.com/compatible-mode/v1"
api_key = "sk-你的阿里云百炼密钥"
client_type = "openai"
max_retry = 2
timeout = 120
retry_interval = 10
阿里云百炼的 API Key 获取：登录 百炼控制台，在 API-KEY 管理中创建。Embedding 模型 text-embedding-v4 价格极低（约0.0007元/千token），且有免费额度。

B3. 将第一个 API Provider 的 name 改为 "BaiLian"

查找

name = "DeepSeek"（默认生成的provider名，也可能是其他名字）

替换为

name = "BaiLian"

这个名字随意，但后续模型定义中的 api_provider 要与之对应。

B4. 修改现有模型定义的 api_provider

确保所有 DeepSeek 模型的 api_provider 指向 "BaiLian"：

每个 [[models]] 段中，查找

api_provider = "DeepSeek"（或其他旧名字）

替换为

api_provider = "BaiLian"

B5. 添加 Embedding 模型和视觉模型定义

在 [[models]] 段的末尾、[model_task_config 之前，追加：

[[models]]
model_identifier = "text-embedding-v4"
name = "text-embedding-v4"
api_provider = "Alibaba"
price_in = 0
price_out = 0
visual = false
force_stream_mode = false

[models.extra_params]

[[models]]
model_identifier = "qwen3.6-max-preview"
name = "qwen3.6-max-preview"
api_provider = "Alibaba"
price_in = 0
price_out = 0
visual = true
force_stream_mode = false

[models.extra_params]

B6. 修改任务模型分配

6a. 视觉模型（必须有，否则Bot无法看图）

找到 [model_task_config.vlm] 段，将 model_list 改为：

model_list = ["qwen3.6-max-preview"]

6b. Embedding模型（必须有，否则记忆系统向量检索失效）

找到 [model_task_config.embedding] 段，将 model_list 改为：

model_list = ["text-embedding-v4"]

Embedding 不配置 = 记忆系统降级为纯关键词匹配，Bot的"回忆"能力大幅下降。默认生成的配置中 embedding model_list 为空，必须手动填入。

6c. 可选：添加多模型负载均衡

如果希望不同任务用不同模型，可以在各 task_config 的 model_list 中添加多个模型：

# replyer 用两个模型随机选择
[model_task_config.replyer]
model_list = ["deepseek-v4-flash", "deepseek-v4-pro"]
selection_strategy = "random"

# planner 也用两个
[model_task_config.planner]
model_list = ["deepseek-v4-flash", "deepseek-v4-pro"]
selection_strategy = "random"

阶段 C：修改 bot_config.toml

文件docker-config/mmc/bot_config.toml

这是Bot的主配置文件，控制人格、聊天行为、WebUI等。

C1. 填写 QQ 号

查找

qq_account = ""（默认为空）

替换为你的Bot QQ号

qq_account = "你的QQ机器人号"

此QQ号需要与 NapCat 登录的QQ号一致。如果你还没登录NapCat，可以先跳过，登录后再回来填。

C2. 修改 WebUI host（Docker部署必须改！）

查找

host = "127.0.0.1"（在 [webui] 段中）

替换为

host = "0.0.0.0"

Docker部署时 webui.host 必须改为 "0.0.0.0"！默认的 "127.0.0.1" 在容器内只监听容器自身，宿主机无法通过端口映射访问WebUI。不改则浏览器打不开 http://localhost:18001。

C3. 自定义 Bot 昵称

查找

nickname = "麦麦"

替换为你想要的昵称

nickname = "你的Bot昵称"

C4. 自定义人格设定（核心灵魂）

查找

personality = "... "（在 [personality] 段中，内容很长）

替换为你自己编写的Bot人设

人设编写要点：

  • 使用第二人称（"你是..."）
  • 建议200字以内，描述核心身份、性格特征、说话习惯
  • 可以加入特殊偏爱（对特定QQ号更友好）
  • 可以加入禁忌（不要做什么）

C5. 自定义表达风格

查找

reply_style = "..."（在 [personality] 段中）

替换为你自己编写的语言风格描述

风格编写要点：描述语气特点、常用表达、表情符号使用习惯、对不同人的态度差异。

C6. 可选：调整聊天参数

[chat]
talk_value = 0.3              # 群聊发言频率（0-1，越小越沉默）
private_talk_value = 1        # 私聊频率
inevitable_at_reply = true    # AT必回
max_context_size = 40         # 上下文长度
enable_context_optimization = true  # 优化上下文消耗（推荐开启）
mid_term_memory = true        # 中期记忆摘要（推荐开启）

阶段 D：NapCat 登录与 Adapter 配置

D1. 登录 NapCat

  1. 浏览器打开 http://localhost:6099
  2. 如果需要token，查看 docker-config/napcat/webui.json 中的 "token" 字段
  3. 扫码登录你的Bot QQ号
  4. 登录成功后记住QQ号，回填到 bot_config.toml 的 qq_account

D2. 配置 NapCat 的 WebSocket Server

登录QQ后，NapCat会自动生成 docker-config/napcat/onebot11_{QQ号}.json。

打开该文件，确认 websocketServers 配置如下：

"websocketServers": [
  {
    "enable": true,
    "name": "maibot",
    "host": "0.0.0.0",
    "port": 3001,
    "reportSelfMessage": false,
    "messagePostFormat": "array",
    "token": "",
    "heartInterval": 30000
  }
]

关键点：enable 必须为 true，port 必须为 3001（与adapter配置一致）。

D3. 配置 NapCat-Adapter

文件data/MaiMBot/plugins/MaiBot-Napcat-Adapter/config.toml

以下三项是最容易出错的地方，务必仔细检查！

3a. 启用插件

查找

enabled = false（在 [plugin] 段中）

替换为

enabled = true

3b. 修改 NapCat 连接地址

查找

host = "127.0.0.1"（在 [napcat_server] 段中）

替换为

host = "napcat"

host 绝对不能是 "127.0.0.1"！在Docker环境中，MaiBot和NapCat是不同容器，127.0.0.1在容器内只代表容器自己。必须使用Docker Compose中的服务名 "napcat"。

3c. 添加群聊白名单

查找

group_list = []（在 [chat] 段中）

替换为你的群号列表

group_list = [你的群号1, 你的群号2]

NapCat适配器默认启用聊天名单过滤，群聊默认白名单模式。群号没写进 group_list，群消息会被直接丢弃，看起来就像连接正常但Bot没反应。

同理，私聊白名单 private_list 也要添加允许私聊的QQ号。

阶段 E：重启并验证

E1. 重启容器让配置生效

docker compose restart core

E2. 查看日志确认

# 查看插件加载状态
docker logs maim-bot-core --since 1m 2>&1 | grep -iE "loaded=|failed=|inactive="

# 确认NapCat连接成功
docker logs maim-bot-core --since 1m 2>&1 | grep -iE "适配器已连接|NapCat"

# 确认Embedding正常
docker logs maim-bot-core --since 1m 2>&1 | grep -iE "嵌入维度|embedding.*降级"

E3. 期望的正常输出

# 插件加载正常（数字因安装数量而异）
Runner 插件初始化完成: loaded=N failed=0 inactive=0

# NapCat连接成功
NapCat 适配器已连接: ws://napcat:3001
NapCat 元事件：Bot 你的QQ号 已建立连接

# Embedding正常（不应有降级警告）
嵌入维度: 1024

E4. 常见错误速查

日志关键词原因修复
重复插件ID两个napcat-adapter冲突确认wrapper脚本已配置
Cannot connect to host 127.0.0.1adapter host未改改为 "napcat"
embedding 请求失败: Not Foundembedding模型不支持改为 text-embedding-v4
embedding 进入降级态embedding未配置或失败检查model_config.toml
配置版本非法插件自身bugWebUI卸载重装该插件
