# MaiBot Docker 运维操作手册

## 常用 Docker 命令

```powershell
# 重启容器（不重建）
docker restart maim-bot-core

# 重建容器（应用 docker-compose.yml 变更）
docker compose up -d core --force-recreate

# 查看日志
docker logs maim-bot-core --tail 100
docker logs maim-bot-core -f  # 实时跟踪

# 容器内执行命令（Windows PowerShell 需设置 MSYS_NO_PATHCONV）
$env:MSYS_NO_PATHCONV=1; docker exec maim-bot-core sh -c '命令'

# 安装依赖到 venv（容器内只用 uv，无 pip）
$env:MSYS_NO_PATHCONV=1; docker exec maim-bot-core uv pip install <包名>

# 查看已安装包
$env:MSYS_NO_PATHCONV=1; docker exec maim-bot-core uv pip list
```

## 日志

### 日志路径

| 环境 | 路径 |
|------|------|
| 容器内 | `/MaiMBot/logs/` |
| 宿主机 | `<项目根>/data/MaiMBot/logs/` |

日志路径为硬编码相对路径 `Path("logs")`，不可配置。Docker 通过卷映射 `./data/MaiMBot/logs:/MaiMBot/logs` 暴露到宿主机。

### 日志文件结构

```
logs/
├── app_YYYYMMDD_HHMMSS.log.jsonl   # 主日志（JSONL格式，5MB轮转）
├── debug_planner_cache/             # 规划器调试缓存
├── debug_reply_cache/               # 回复调试缓存
├── llm_cache_stats/                 # LLM缓存统计
├── llm_request/                     # LLM请求快照
├── maisaka_prompt/                  # Maisaka提示词预览
│   ├── replyer/                     # 回复器提示词
│   │   └── qq_group_*/             # 按群/私聊分目录
│   │       ├── <timestamp>.html     # HTML格式（可浏览器查看）
│   │       └── <timestamp>.txt      # 纯文本格式
│   └── planner/                     # 规划器提示词
├── maisaka_reply_effect/            # 回复效果评估
├── expression_review/               # 表达审核
└── plugin_runtime_debug/            # 插件运行时调试
```

### 日志轮转与清理

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `log_file_max_bytes` | 5MB | 单文件最大大小，超过轮转 |
| `max_log_files` | 30 | 最多保留文件数 |
| `log_cleanup_days` | 30 | 日志保留天数 |

配置位于 `bot_config.toml` 的 `[log]` 段。

### 搜索日志文件（Everything）

使用 [Everything](https://www.voidtools.com/) 的命令行工具 `es.exe` 搜索：

```powershell
# 先确保 Everything 后台服务在运行
Start-Process "C:\Program Files\Everything\Everything.exe" -WindowStyle Minimized

# 搜索文件夹
& "d:\es.exe" folder:qq_group_*

# 搜索指定路径下
& "d:\es.exe" folder:qq_group_* | path:maisaka_prompt

# 用 Everything GUI 搜索
& "C:\Program Files\Everything\Everything.exe" -s "folder:qq_group_*"
```

**注意**：
- `es.exe` 依赖 Everything 后台服务运行，否则报 `Error 8: Everything IPC window not found`
- PowerShell 中调用可执行文件必须用 `&` 操作符：`& "path\es.exe" 参数`
- `es.exe` 输出到控制台，`Everything.exe -s` 在 GUI 窗口显示

## 配置文件

| 文件 | 宿主机路径 | 容器内路径 | 修改方式 |
|------|-----------|-----------|---------|
| bot_config.toml | `docker-config/mmc/bot_config.toml` | `/MaiMBot/config/bot_config.toml` | 直接编辑，restart 生效 |
| model_config.toml | `docker-config/mmc/model_config.toml` | `/MaiMBot/config/model_config.toml` | 直接编辑，restart 生效 |
| 插件代码 | `data/MaiMBot/plugins/` | `/MaiMBot/plugins/` | 直接编辑，restart 生效 |

## 排障

| 问题 | 排查方式 |
|------|---------|
| 容器反复重启 | `docker logs maim-bot-core --tail 50` 看 ImportError/NameError |
| 插件 LLM 400 错误 | 检查强制思考模型不能设 `enable_thinking=False` |
| HookHandler 超时 | 涉及 LLM/嵌入的 Hook 需设 `timeout_ms=30000` |
| DeepSeek V4 thinking 格式 | TOML 中必须用对象格式 `[models.extra_params.thinking] type = "disabled"` |
| Pydantic extra=forbid 丢弃配置 | 在 wrapper 中添加对应字段定义 |
